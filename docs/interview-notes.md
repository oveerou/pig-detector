# 面试讲解笔记

## 1. 为什么做这个项目

针对牧原算法岗，选择智慧养殖中可见光猪只检测与计数分析方向。该方向数据易获取、任务明确、展示直观，是养殖场景智能化的重要切入点。

## 2. 数据和任务

- 训练图片：700 张
- 训练标注：700 个 JSON 文件
- 测试图片：220 张
- 任务：猪只目标检测、数量统计、密度分析、拥挤风险提示

## 3. 技术路线

```text
JSON 标注审计 → JSON 转 YOLO 格式 → 数据集划分 → YOLOv8n 训练 → 单图推理 → 批量分析 → Gradio 演示
```

## 4. 为什么用 YOLOv8n

- 模型轻量，训练和推理速度快
- 适合在资源受限环境下部署原型
- Ultralytics 框架封装完善，便于快速验证

## 5. 风险提示怎么设计

基于三个维度做规则化判断：

- 猪只数量：≥8 中等风险，≥15 高风险
- 检测框面积占比：≥25% 中等风险，≥45% 高风险
- 平均置信度：＜0.5 提示可能存在遮挡或图像质量问题

## 6. 目前边界

- 使用本地样例数据，非牧原真实生产数据
- 不包含体重估计、行为跟踪、红外体温、声音识别
- 第一版为原型系统，未做生产级部署

## 7. 后续优化方向

- 视频帧连续检测与 ByteTrack 跟踪
- 猪只行为识别（躺卧、进食、打斗）
- 体重估计与体尺测量
- 多模态融合（可见光 + 红外 + 声音）
